# ليبيكا ERP — Mobile App Build Guide

## Prerequisites

| Tool | Version | Download |
|------|---------|----------|
| Node.js | ≥ 18 | nodejs.org |
| Android Studio | Latest | developer.android.com/studio |
| Java JDK | 17+ | adoptium.net |
| (iOS only) macOS + Xcode | ≥ 15 | App Store |

---

## 1. First-time setup

```bash
cd "mobile-app"
npm install

# Copy .env.example → .env and set your backend URL
copy .env.example .env
```

Edit `.env`:
```
VITE_API_URL=http://YOUR_SERVER_IP:8001
```
> On Android emulator use `http://10.0.2.2:8001` to reach host machine.  
> On a real device, use your machine's LAN IP e.g. `http://192.168.1.x:8001`.

---

## 2. Run as PWA in the browser (fastest dev cycle)

```bash
# Start FastAPI backend (in ERP mobile/ folder)
python -m uvicorn app:app --host 0.0.0.0 --port 8001 --reload

# In another terminal, start React dev server
cd mobile-app
npm run dev
```

Open **http://localhost:5173** on your phone's browser (same WiFi network).  
Add to home screen from the browser menu → behaves like a native app.

---

## 3. Build production PWA

```bash
npm run build
# Output in mobile-app/dist/
```

Host the `dist/` folder on any static server (Nginx, Vercel, Netlify, etc.).

---

## 4. Build Android APK

### 4a. Install Android Studio
Download from https://developer.android.com/studio and install it.  
Open **SDK Manager** → install **Android SDK Platform 34**.

### 4b. Sync Capacitor

```bash
npm run cap:sync
# This builds React, then syncs web assets into android/
```

### 4c. Open in Android Studio

```bash
npx cap open android
```

Android Studio will open `mobile-app/android/`.  
Wait for Gradle sync to finish (first time takes a few minutes).

### 4d. Generate debug APK (for testing)

In Android Studio:  
**Build → Build Bundle(s) / APK(s) → Build APK(s)**

The APK will be at:
```
android/app/build/outputs/apk/debug/app-debug.apk
```

Transfer to your Android device and install  
(Settings → Security → Allow from unknown sources).

### 4e. Generate signed release APK (for distribution)

**Step 1 — Create a keystore (one time only):**
```bash
keytool -genkey -v -keystore libyca-release.jks \
  -alias libyca -keyalg RSA -keysize 2048 -validity 10000
```
Keep `libyca-release.jks` safe — losing it means you can't update the app.

**Step 2 — Add signing config to `android/app/build.gradle`:**
```groovy
android {
    ...
    signingConfigs {
        release {
            storeFile file('../../libyca-release.jks')
            storePassword 'YOUR_STORE_PASSWORD'
            keyAlias 'libyca'
            keyPassword 'YOUR_KEY_PASSWORD'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}
```

**Step 3 — Build release APK:**
In Android Studio: **Build → Generate Signed Bundle / APK → APK → Release**

Or via command line:
```bash
cd android
./gradlew assembleRelease
```
Output: `android/app/build/outputs/apk/release/app-release.apk`

## 5. iOS (PWA — no Mac required)

Open Safari on iPhone/iPad → navigate to your hosted PWA URL →  
tap **Share → Add to Home Screen**.

The app icon appears on the home screen and runs full-screen (standalone mode).

### iOS native IPA (requires macOS + Xcode)

```bash
npx cap add ios
npm run cap:ios     # Opens Xcode
```

In Xcode: select your development team, connect your iPhone, press **Run**.  
For App Store distribution, use **Product → Archive**.

---

## 6. Updating the app

```bash
# After changing React code:
npm run build
npx cap sync

# Then rebuild APK in Android Studio or run:
cd android && ./gradlew assembleRelease
```

---

## 7. Configure backend URL for production

In `.env`:
```
VITE_API_URL=https://your-production-domain.com
```

Then rebuild:
```bash
npm run build && npx cap sync
```

The app will use the production API.  
JWT tokens are stored in localStorage (persisted across app restarts).

```
├── android/              ← Capacitor Android project (open in Android Studio)

│   ├── App.jsx           ← Router + protected routes

├── dist/                 ← Built web assets (generated by npm run build)
├── vite.config.js        ← Vite + PWA plugin config
├── capacitor.config.json ← Capacitor config
└── .env                  ← VITE_API_URL=http://...
```

## 9. Offline behaviour

The Workbox service worker (auto-generated at build time) provides:

- **Static assets** (JS, CSS, HTML): cached on first load, served from cache
- **API GET calls**: NetworkFirst strategy — tries server, falls back to cached response
- **Offline banner**: shown automatically when device loses connectivity
- **POST/PUT/DELETE**: always require network (not cached)

## 10. Troubleshooting

| Problem | Fix |
|---------|-----|
| CORS error in dev | Make sure FastAPI is running on 8001 and Vite proxy is configured |
| API calls fail on device | Use LAN IP in `.env`, not `localhost` |
| Blank screen on Android | Check that `VITE_API_URL` is set correctly |
| Gradle sync fails | File → Invalidate Caches in Android Studio |
| APK won't install | Enable "Install from unknown sources" in Android settings |
